# Installation

Nyra is a **copy-paste** library. Components live in your `components/ui/` directory after you add them — there's no runtime dependency on `nyra`, no provider, no compile-time magic. Once a component is in your project, you own it.

There are two ways to add components:

1. **The Nyra CLI** — one command per component. Recommended.
2. **Manual** — copy the source from the catalog and install the deps yourself.

Both leave you with the same files in the same place.

## Quick start

```bash
pnpm dlx nyra@latest init
pnpm dlx nyra@latest add magnetic-button
```

`init` is run once per project: it writes `lib/utils.ts`, adds the Tailwind preset import to your global CSS, and offers to install the shared runtime deps. `add` is run per component.

For the full CLI reference (commands, flags, Pro login, environment overrides), see [CLI](/docs/cli).

## Prerequisites

Nyra targets the modern Next.js / Tailwind v4 stack. If you're starting fresh:

```bash
pnpm create next-app@latest my-app --typescript --tailwind --app
cd my-app
```

Tailwind **v4 is required** — Nyra uses the v4 `@theme` directive and CSS-first config. If your project is still on v3, upgrade before installing (or see the manual path below for an inline alternative).

<Callout type="info" title="Other frameworks">
  The CLI doesn't care what framework you're in — it only needs a `package.json` and a `components/ui/` directory. We've tested it with Next.js, Vite + React, Remix, and Astro.
</Callout>

## Path A — With the Nyra CLI

### 1. Initialise the project

```bash
pnpm dlx nyra@latest init
```

This is idempotent — running it twice is a no-op. It will:

- Write `lib/utils.ts` (the `cn()` helper used by every component)
- Add `@import "@nyra/tailwind-preset";` to your `app/globals.css`, right after the Tailwind import
- Offer to install: `motion`, `clsx`, `tailwind-merge`, `lucide-react`, `@nyra/tailwind-preset`

### 2. Add components

```bash
pnpm dlx nyra@latest add magnetic-button aurora-background tilt-card
```

Each slug lands in `components/ui/<category>/<name>.tsx`. The CLI aggregates dependency requirements across every component you're adding and prompts once.

### 3. Use the component

```tsx
import { MagneticButton } from "@/components/ui/button/magnetic-button";

export default function Page() {
  return <MagneticButton>Hover me</MagneticButton>;
}
```

That's the whole loop. Repeat step 2 every time you want a new component.

### Pro components

If you have a Pro license, save the key once:

```bash
pnpm dlx nyra@latest login --key NYRA-XXXX-XXXX-XXXX
pnpm dlx nyra@latest add floating-dock
```

The key is stored in `~/.nyra/credentials` (`0600` mode). The registry validates it before returning Pro source. See [CLI · login](/docs/cli) for the full flow.

## Path B — Manual

If you'd rather not run a CLI, the setup is five small steps.

### 1. The base stack

```bash
pnpm create next-app@latest my-app --typescript --tailwind --app
cd my-app
```

### 2. Required runtime dependencies

```bash
pnpm add motion clsx tailwind-merge lucide-react
pnpm add -D @nyra/tailwind-preset
```

Replace `pnpm` with `npm` / `yarn` / `bun` if needed — the dep names don't change.

<Callout type="info" title="Per-component dependencies">
  Each component's detail page lists exactly which packages it needs. Pages without animation often have **zero** extra deps beyond the shared `cn()` helper.
</Callout>

### 3. The `cn` helper

Create `lib/utils.ts`:

```tsx
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

### 4. The Tailwind preset

In your `app/globals.css`, **after** the Tailwind import:

```css
@import "tailwindcss";
@import "@nyra/tailwind-preset";
```

That gives you every Nyra design token (`--background`, `--accent`, `--border`, …) and the Tailwind utilities that read from them (`bg-background`, `text-accent`, `border-border`). Override any variable to rebrand the library — see [Theming](/docs/theming).

### 5. Copy a component

Open the [catalog](/components), pick a component, and copy its source into `components/ui/<category>/<name>.tsx`. Then import and use:

```tsx
import { MagneticButton } from "@/components/ui/button/magnetic-button";

export default function Page() {
  return <MagneticButton>Hover me</MagneticButton>;
}
```

You now own the source — edit freely.

## Project layout

After your first `add`, your project will look like this (paths matter for the default Next.js path alias `@/*`):

```
my-app/
├── app/
│   ├── globals.css           # has @import "@nyra/tailwind-preset"
│   └── page.tsx
├── components/
│   └── ui/
│       ├── button/
│       │   └── magnetic-button.tsx
│       ├── background/
│       │   └── aurora-background.tsx
│       └── card/
│           └── tilt-card.tsx
├── lib/
│   └── utils.ts              # cn() helper
└── package.json
```

If your project uses Next.js's `src/` directory, every path above moves under `src/` — the CLI detects this automatically.

## Updating

There's no `nyra` package to upgrade — components are source files in your repo. To pick up an updated version of a component, re-run `add` with `--force`:

```bash
pnpm dlx nyra@latest add magnetic-button --force
```

This overwrites the file. Diff first if you've customised it.
