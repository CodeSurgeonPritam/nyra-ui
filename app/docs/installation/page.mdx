# Installation

There are two ways to add Nyra to a project. Both leave the source in your `components/ui/` directory — Nyra is copy-paste, not a runtime dependency.

## With the Nyra CLI (recommended)

The CLI handles the setup, fetches components on demand, and gates Pro components behind your license key.

```bash
pnpm dlx nyra@latest init
```

That command:

1. Detects your package manager (pnpm, yarn, bun, or npm)
2. Writes `lib/utils.ts` (the `cn()` helper)
3. Adds `@import "@nyra/tailwind-preset";` to your global stylesheet
4. Offers to install the runtime deps: `motion`, `clsx`, `tailwind-merge`, `lucide-react`, `@nyra/tailwind-preset`

Then pick components from the [catalog](/components) and pull them in:

```bash
pnpm dlx nyra@latest add magnetic-button aurora-background tilt-card
```

Each component lands in `components/ui/<category>/<name>.tsx` and the CLI prompts to install any missing deps.

### Pro components

```bash
pnpm dlx nyra@latest login --key NYRA-XXXX-XXXX-XXXX
pnpm dlx nyra@latest add floating-dock
```

`login` saves your key to `~/.nyra/credentials` (file mode `0600`). Subsequent `add` calls send it as `Authorization: License NYRA-…`; the registry validates against the active license table before returning source.

## Manual install

If you'd rather not run a CLI, the setup is five small steps.

### 1. The base stack

```bash
pnpm create next-app@latest my-app --typescript --tailwind --app
cd my-app
```

### 2. Required runtime dependencies

```bash
pnpm add motion clsx tailwind-merge lucide-react
pnpm add -D @nyra/tailwind-preset
```

<Callout type="info" title="Per-component dependencies">
  Each component's detail page lists exactly which packages it needs. Pages without animation often have **zero** extra deps beyond the shared `cn()` helper.
</Callout>

### 3. The `cn` helper

Create `lib/utils.ts`:

```tsx
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

### 4. The Tailwind preset

In your `app/globals.css`, **after** the Tailwind import:

```css
@import "tailwindcss";
@import "@nyra/tailwind-preset";
```

That gives you every Nyra design token (`--background`, `--accent`, `--border`, …) and the Tailwind utilities that read from them (`bg-background`, `text-accent`, `border-border`). Override any variable to rebrand the library — see [Theming](/docs/theming).

### 5. Copy a component

Open the [catalog](/components), pick a component, copy its source into `components/ui/<category>/<name>.tsx`, and use it like any other React component:

```tsx
import { MagneticButton } from "@/components/ui/button/magnetic-button";

export default function Page() {
  return <MagneticButton>Hover me</MagneticButton>;
}
```

You now own the source — edit freely, or leave it alone for the next update.
