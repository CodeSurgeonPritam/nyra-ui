# Components

Nyra organizes the catalog into eight categories. Each category lives in its own folder under `components/ui/` so you can drop a category into your project without picking through a flat list.

## Folder layout

```bash
components/ui/
├── background/   # Drifting gradients, particles, beams
├── button/       # Magnetic, shimmer, beam variants
├── card/         # Tilt, glow, bento, hover lifts
├── data/         # Marquees, comparison tables, pricing
├── form/         # Multi-step, autocomplete, animated inputs
├── hero/         # Hero sections with motion baked in
├── nav/          # Floating docks, animated tabs, sidebars
└── text/         # Word reveals, tickers, gradient text
```

## What every component ships

Each component file is a single self-contained `.tsx` file. It exports a named React component (e.g. `MagneticButton`), declares its props with TypeScript, and uses `cn()` for class composition. No imports outside of:

- `react`
- `motion` (animated components only)
- `lucide-react` (icons, when used)
- `@/lib/utils` (the `cn` helper)

That's it. Copy the file, read the props off the TS types, ship.
